# LTS (Long-Term Support) 정책

## 1. 개요

AuthFusion SSO Platform의 장기지원(LTS) 정책을 정의한다.
CC 인증 평가 이후 안정적인 유지보수를 보장하기 위한 정책이다.

## 2. LTS 브랜치

| 브랜치 | 대상 | 지원 기간 | 지원 범위 |
|--------|------|----------|----------|
| `release/sso-1.x-lts` | TOE (CC 인증 배포물) | CC 인증 유효 기간 | 보안패치 + 버그수정 |
| `release/platform-1.x` | KeyHub/Connect 확장 | 1년 | 기능 추가 + 보안패치 |

## 3. 백포트 규칙

### 3.1 LTS 브랜치 허용 변경
- 보안 취약점 패치 (CVE 수정)
- 심각한 버그 수정 (데이터 손실, 인증 우회)
- 의존성 보안 업데이트

### 3.2 LTS 브랜치 금지 변경
- 새 기능 추가
- API 변경 (호환성 깨지는 변경)
- 리팩토링 (기능 변경 없는 코드 정리)
- 성능 최적화 (안정성 우선)

### 3.3 백포트 프로세스
1. `main` 브랜치에서 먼저 수정
2. Cherry-pick으로 LTS 브랜치에 백포트
3. PR 생성 (최소 2인 코드 리뷰)
4. CC 영향 분석 라벨 필수
5. CI + 보안 스캔 통과

## 4. 버전 태그

| 패턴 | 대상 | 예시 |
|------|------|------|
| `sso-v*` | TOE 릴리즈 | `sso-v1.2.3` |
| `keyhub-v*` | KeyHub 릴리즈 | `keyhub-v0.9.1` |
| `connect-v*` | Connect 릴리즈 | `connect-v0.5.0` |

## 5. 지원 종료 (EOL) 절차

1. EOL 6개월 전 공지
2. EOL 3개월 전 마지막 보안 패치
3. EOL 이후 보안 패치 미제공
4. 마이그레이션 가이드 제공
