# 브랜치 보호 정책

## 브랜치 전략

| 브랜치 | 용도 | 보호 수준 |
|--------|------|-----------|
| `main` | 다음 메이저/마이너 개발 | 필수 PR 리뷰, CI 통과 |
| `release/sso-1.x-lts` | TOE LTS 브랜치 (백포트만) | 필수 PR 리뷰 2인, CI + 보안스캔 통과 |
| `release/platform-1.x` | KeyHub/Connect 확장 브랜치 | 필수 PR 리뷰, CI 통과 |

## 태그 규칙

| 패턴 | 대상 | 예시 |
|------|------|------|
| `sso-v*` | TOE 릴리즈 | `sso-v1.2.3` |
| `keyhub-v*` | KeyHub 플랫폼 릴리즈 | `keyhub-v0.9.1` |
| `connect-v*` | Connect 플랫폼 릴리즈 | `connect-v0.5.0` |

## main 브랜치 보호 규칙

- PR을 통한 머지만 허용 (직접 push 금지)
- 최소 1인 이상 코드 리뷰 승인 필수
- CI 빌드 통과 필수
- 보안 스캔 통과 필수
- 브랜치 최신 상태 유지 필수

## TOE LTS 브랜치 보호 규칙

- 최소 2인 이상 코드 리뷰 승인 필수
- TOE 변경 시 CC 영향 분석 라벨 필수
- 보안 취약점 패치 및 버그 수정만 백포트 허용
- 기능 추가 금지
