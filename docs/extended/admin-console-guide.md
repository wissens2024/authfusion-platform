# Admin Console 사용 가이드

## AuthFusion Admin Console (Next.js) 관리자 웹 콘솔 가이드

---

## 1. 개요

AuthFusion Admin Console은 SSO Server를 관리하기 위한 웹 기반 관리자 인터페이스이다.
Next.js 14 기반으로 개발되었으며, SSO Server API를 통해 데이터에 접근한다 (직접 DB 접근 없음).

### 1.1 기술 스택

| 기술 | 버전 | 용도 |
|------|------|------|
| Next.js | 14.x | 웹 프레임워크 |
| React | 18.x | UI 라이브러리 |
| TypeScript | 5.x | 타입 안전성 |
| Tailwind CSS | 3.x | 스타일링 |

### 1.2 접속 정보

- **URL**: `https://{host}:3000`
- **인증**: SSO Server 로그인 (ADMIN 역할 필수)
- **지원 브라우저**: Chrome 120+, Firefox 120+, Edge 120+, Safari 17+

---

## 2. 로그인

### 2.1 로그인 절차

1. Admin Console URL에 접속
2. SSO Server 로그인 페이지로 리다이렉트
3. 관리자 자격증명 입력 (사용자명, 비밀번호)
4. 인증 성공 시 Admin Console 대시보드로 리다이렉트
5. `ADMIN` 역할이 아닌 경우 접근 거부

### 2.2 세션 관리

- Admin Console 세션은 SSO Server 세션과 연동
- 세션 타임아웃: CC 모드 30분 (비활성 기준)
- 타임아웃 시 자동 로그아웃 및 로그인 페이지로 리다이렉트

---

## 3. 대시보드

### 3.1 대시보드 개요

로그인 후 표시되는 메인 화면으로, 시스템 현황을 한눈에 파악할 수 있다.

### 3.2 주요 지표 카드

| 지표 | 설명 | 데이터 소스 |
|------|------|-----------|
| **활성 사용자 수** | 현재 활성 상태의 사용자 수 | `/api/v1/users?enabled=true` |
| **활성 세션 수** | 현재 활성 SSO 세션 수 | `/api/v1/sessions?status=ACTIVE` |
| **등록 클라이언트 수** | 등록된 OAuth2 클라이언트 수 | `/api/v1/clients` |
| **오늘 로그인 수** | 금일 로그인 성공 횟수 | `/api/v1/audit/statistics` |
| **로그인 실패 수** | 금일 로그인 실패 횟수 | `/api/v1/audit/statistics` |
| **계정 잠금 수** | 금일 계정 잠금 발생 횟수 | `/api/v1/audit/statistics` |

### 3.3 차트 영역

- **로그인 추이**: 최근 24시간 로그인 성공/실패 추이 차트 (시간별)
- **토큰 발급 추이**: 최근 24시간 토큰 발급 건수 추이 차트
- **이벤트 유형 분포**: 감사 이벤트 유형별 비율 도넛 차트

### 3.4 최근 활동 목록

최근 발생한 보안 이벤트 10건을 시간순으로 표시:
- 로그인 성공/실패
- 계정 잠금
- 세션 생성/종료
- 설정 변경

## 4. 사용자 관리

### 4.1 사용자 목록

좌측 메뉴 **사용자 관리**를 클릭하면 사용자 목록이 표시된다.

#### 기능

- **검색**: 사용자명, 이메일, 이름으로 검색
- **필터**: 활성/비활성, 역할별 필터링
- **정렬**: 생성일, 이름, 최근 로그인 순
- **페이징**: 페이지당 10/20/50건 선택

#### 목록 컬럼

| 컬럼 | 설명 |
|------|------|
| 사용자명 | 로그인 ID |
| 이름 | 표시 이름 |
| 이메일 | 이메일 주소 |
| 역할 | 할당된 역할 배지 |
| 상태 | 활성/비활성/잠금 |
| 최근 로그인 | 마지막 로그인 일시 |
| 생성일 | 계정 생성 일시 |

### 4.2 사용자 생성

1. **새 사용자** 버튼 클릭
2. 사용자 정보 입력:
   - 사용자명 (필수, 고유)
   - 비밀번호 (필수, 비밀번호 정책 준수)
   - 이메일 (필수, 고유)
   - 이름 (필수)
   - 역할 선택 (기본: USER)
   - 초기 로그인 시 비밀번호 변경 강제 여부
3. **생성** 버튼 클릭

### 4.3 사용자 상세/수정

사용자 목록에서 사용자를 클릭하면 상세 정보가 표시된다.

#### 수정 가능 항목

- 이름
- 이메일
- 역할
- 활성/비활성 상태

#### 관리 작업

- **비밀번호 초기화**: 임시 비밀번호 발급
- **계정 잠금 해제**: 무차별 대입 공격으로 잠긴 계정 해제
- **세션 강제 종료**: 해당 사용자의 모든 활성 세션 종료
- **계정 비활성화**: 계정 비활성화 (로그인 불가)

## 5. 클라이언트 관리

### 5.1 클라이언트 목록

좌측 메뉴 **클라이언트 관리**를 클릭하면 등록된 OAuth2 클라이언트 목록이 표시된다.

#### 목록 컬럼

| 컬럼 | 설명 |
|------|------|
| 클라이언트 이름 | 애플리케이션 식별 이름 |
| Client ID | OAuth2 클라이언트 식별자 |
| 유형 | CONFIDENTIAL / PUBLIC |
| 스코프 | 허용된 스코프 목록 |
| 등록일 | 클라이언트 등록 일시 |

### 5.2 클라이언트 등록

1. **새 클라이언트** 버튼 클릭
2. 필수 정보 입력 (상세 설정은 클라이언트 관리 가이드 참조)
3. **등록** 클릭
4. 생성된 Client ID와 Client Secret을 확인하고 안전하게 보관

> **중요**: Client Secret은 이 화면에서만 확인할 수 있다. 화면을 닫으면 다시 확인할 수 없으며, 분실 시 재생성해야 한다.

### 5.3 클라이언트 상세/수정

클라이언트 목록에서 항목을 클릭하면 상세 정보와 수정 화면이 표시된다.

---

## 6. 역할 관리

### 6.1 역할 목록

좌측 메뉴 **역할 관리**를 클릭하면 RBAC 역할 목록이 표시된다.

### 6.2 역할 생성/수정

1. **새 역할** 또는 기존 역할의 **편집** 클릭
2. 역할명, 설명 입력
3. 권한 체크박스에서 부여할 권한 선택
4. **저장** 클릭

### 6.3 역할 할당

- 사용자 상세 페이지에서 **역할** 섹션의 **편집** 클릭
- 할당할 역할을 선택/해제
- **저장** 클릭

---

## 7. 세션 관리

### 7.1 세션 목록

좌측 메뉴 **세션 관리**를 클릭하면 활성 세션 목록이 표시된다.

#### 목록 컬럼

| 컬럼 | 설명 |
|------|------|
| 사용자 | 세션 소유 사용자 |
| IP 주소 | 접속 IP |
| 브라우저 | User-Agent 정보 |
| 상태 | ACTIVE / EXPIRED / TERMINATED |
| 생성 시간 | 세션 생성 일시 |
| 마지막 활동 | 최근 활동 일시 |
| 만료 예정 | 세션 만료 예정 시간 |

### 7.2 세션 강제 종료

- 개별 세션: 세션 행의 **종료** 버튼 클릭
- 사용자 전체: 사용자 필터 후 **전체 종료** 클릭

---

## 8. 감사 로그

### 8.1 감사 로그 조회

좌측 메뉴 **감사 로그**를 클릭하면 보안 이벤트 목록이 표시된다.

#### 필터 옵션

- **이벤트 유형**: 로그인 성공/실패, 토큰 발급/폐기, 세션 관련, 관리 작업 등
- **사용자**: 특정 사용자의 이벤트만 조회
- **날짜 범위**: 시작일 ~ 종료일
- **IP 주소**: 특정 IP에서 발생한 이벤트

#### 목록 컬럼

| 컬럼 | 설명 |
|------|------|
| 일시 | 이벤트 발생 시간 |
| 이벤트 유형 | AUTH_LOGIN_SUCCESS 등 |
| 사용자 | 관련 사용자 |
| IP 주소 | 요청 IP |
| 상세 | 이벤트 상세 내용 |

### 8.2 감사 통계

**감사 로그** 메뉴의 **통계** 탭에서 집계된 통계를 확인할 수 있다:

- 시간별/일별/주별 이벤트 추이
- 이벤트 유형별 분포
- 사용자별 활동 순위
- 클라이언트별 토큰 발급 현황

### 8.3 감사 로그 내보내기

- **CSV 내보내기**: 조회 결과를 CSV 파일로 다운로드
- 최대 내보내기 건수: 10,000건

## 11. 문제 해결

### 11.1 로그인 불가

- SSO Server가 실행 중인지 확인
- ADMIN 역할이 할당되어 있는지 확인
- 계정이 잠겨 있지 않은지 확인
- 브라우저 캐시/쿠키 삭제 후 재시도

### 11.2 데이터가 표시되지 않음

- SSO Server API 연결 상태 확인
- 브라우저 개발자 도구에서 네트워크 오류 확인
- 토큰 만료 여부 확인 (자동 갱신 실패 시 재로그인)

### 11.3 Admin Console 접속 불가

- Admin Console 프로세스 실행 확인 (`npm run dev` 또는 PM2)
- 포트 3000 방화벽 규칙 확인
- 환경 변수 `NEXT_PUBLIC_SSO_SERVER_URL` 설정 확인
